# Contributing (Chironte)

## Flujo estándar
1) Crear Issue (Bug/Feature/Task)
2) Mover a **Ready** cuando esté refinada (criterios + owner)
3) Crear rama desde la issue (si aplica)
4) Abrir PR con `Closes #ID`
5) Review → Merge → Issue en Done

## Convenciones
- Commits: claros y pequeños
- PRs: describir qué/por qué + evidencia
- Si es DB: adjuntar plan/impacto

## Definition of Done (DoD)
- Criterios de aceptación cumplidos
- PR mergeado
- Sin CI rojo
- Documentación/README actualizado si aplica
